Creating a new branch

  1. Open the source control tab and click on the “Git Graph”. (If you don’t have the Git Graph icon, install the Git Graph VSCode extension)
  2. On the right side of the screen, click on the cloud icon and then the refresh icon. This will make sure you have all the current updates
  3. Make sure you are on the master | origin branch. (If you don’t see a circle next to the master branch, right-click on the master branch and click “Checkout Branch”)
  4. Now click on the 3 dots next to the Git Graph button and select “Checkout to…”
  5. A prompt will show up at the top of VSCode. Select “+ Create new branch…”
  6. Enter the name of what you want to call the branch and press “Enter”. The naming convention we have been following is website/main-purpose-of-branch
  7. You should now be on your new branch! When you are done making the code changes you need to, commit the changes and publish the branch (See below for steps)

Committing Changes and Publishing Branch

  1. Once you’ve made the changes you needed to make in the codebase, go to the “Source Control” tab. You should see all the files you made changes to.
  2. Enter a message for the commit and then click “Commit”
  3. After clicking the “Commit” button, it will change to a “Publish Branch” button. Click “Publish Branch”. (This step might take a few seconds)
  4. Next, go to the Dev Bros HQ github. You should see a yellow banner with the name of your branch and a button that says “Compare & pull request”. Click “Compare & pull request”.
  5. You will be navigated to the “Open a pull request” page. Add additional comments if you prefer, then click “Create pull request”.
  6. Next, tell Rico that you just created a pull request and the branch name of the pull request and he will review the code. You will either get feedback or the pull request will be merged into the master branch and your code will be live!
